# 人間の役割 · Planning Gate（Step 02）

Planning Gate段階におけるヒューマンレビュアーの役割を定義いたします。TDDアンロック前にスコープ・アーキテクチャ・工数を確定する責任を担います。

## WHO（誰が）

| 役割 | 責務 |
|---|---|
| **テックリード** (Long N.) | アーキテクチャ · 技術的実現可能性 · 依存リスク |
| **PM** (Quân T.) | スコープ · スケジュール · リソース配分 |
| **PO**（任意） | ビジネス優先度の整合性確認 |

## WHAT（何を検証するか）

- [ ] 計画はPRDのスコープに合致しておりますか?
- [ ] アーキテクチャ判断は妥当でございますか?
- [ ] 12週間の工数は現実的でしょうか?
- [ ] リスクは十分に特定 + 緩和されておりますか?
- [ ] クリティカルパスは正しいでしょうか（KYC → Auth → Bank → Transfer）?
- [ ] 依存関係（Sepay、VNPT）の契約は準備完了でしょうか?

## WHEN（いつ承認するか）

- 6つのチェックリスト全項目 ✓
- ブロッキング懸念がゼロ
- レビュアーがサインオフ

## WHEN（いつ却下するか）

- スコープの大きな乖離 → Planner Agentが再修正
- アーキテクチャ懸念 → Researcherがドメインを再調査
- 工数見積もりが30%以上乖離 → 追加調査の上で再ベースライン

## SLA

- 計画がレディとなってから**48時間以内**にレビュー
- リビジョンサイクル: ステアリングコミッティへエスカレーション前に最大2ラウンド
