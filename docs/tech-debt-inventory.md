## 高重要度の詳細

### TD-01 · DBスキーマ共有
- **問題点:** wallet-serviceが`transfers`テーブルへ直接SQL → 密結合
- **計画:** ストラングラーフィグマイグレーション W3〜W7
- **切替時の緩和策:** 両スキーマへデュアルライト、整合性をモニタリング

### TD-02 · Wallet神モジュール
- **問題点:** 18k LOC、台帳 + 残高キャッシュ + アナリティクスが混在
- **計画:** `ledger-service`を抽出 + wallet-serviceはファサードとして残す
- **リスク:** 内部呼び出し元への破壊的変更（12サービスがwalletに接続）

### TD-03 · トップアップ冪等性
- **問題点:** idem-keyなしリトライ → 二重課金の可能性
- **計画:** `Idempotency-Key`ヘッダ追加（24h Redis SETNXガード）
- **後方互換:** ヘッダ欠落時はUUIDフォールバック + 警告ログ
