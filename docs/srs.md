**ステータス:** プロダクトオーナー・コンプライアンス・エンジニアリングリードによる承認済み

## 1. はじめに

### 1.1 目的
本SRSドキュメントは、VietPay MVPの要件を定義いたします。VietPayは、ベトナム市場向けのP2P送金およびVietQR決済に特化した電子ウォレットとなります。

### 1.2 スコープ
**対象範囲（MVP）:**
- サインアップ + eKYC（CMND/CCCDスキャン + セルフィーリブネス検証）
- 銀行口座連携（Sepay経由でベトナム主要10行に対応）
- P2P送金（電話番号 / QR / 銀行口座）
- VietQR決済（生成 + スキャン）
- フィルタ付きトランザクション履歴
- プッシュ通知（FCM + APNs）
- 生体認証（Face ID / 指紋認証）

**対象範囲外（後続フェーズ）:**
- 公共料金支払い / モバイルチャージ（フェーズ2）
- 目標型貯蓄機能（フェーズ2）
- 投資商品（フェーズ3）
- Webアプリ（フェーズ3）

### 1.3 用語定義
- **VietQR:** ベトナム国家QR決済規格（Napas運営）
- **eKYC:** 電子的本人確認（electronic Know Your Customer）
- **SBV（ベトナム国家銀行）:** State Bank of Vietnam
- **Idempotency Key:** 各トランザクションリクエストに付与されるユニークなUUID。二重支払い防止用となります
- **ウォレット:** VietPay内のユーザーVND残高（独立した元帳）

## 2. 機能要件

### FR-001 — ユーザー登録
**優先度:** P0（MVP）· **出典:** US-001
- ベトナム電話番号（+84）でのユーザー登録
- 6桁OTP、有効期間5分
- トランザクション用6桁PINの設定
- 生体認証の有効化（任意。オンボーディング時に推奨）

**受入基準:**
- ベトナム全キャリア対応（Viettel、VinaPhone、MobiFone、Vietnamobile）
- レート制限：1番号あたり3 OTP / 時間
- PINは連続値（123456）または同一値の繰り返し（111111）を許可しません

### FR-002 — eKYC本人確認
**優先度:** P0 · **出典:** US-002、SBV通達23/2019
- CMNDまたはCCCDのスキャン（表面 + 裏面）
- リブネス検出付きセルフィー（瞬き / 顔の向き変更）
- VNPT eKYC API経由でのOCR + 顔照合
- 信頼度が90%未満の場合は手動レビューキューへ

**受入基準:**
- 自動承認率92%以上（手動レビューなし）
- KYCデータはAES-256-GCMによる保管時暗号化
- KYC記録は5年間保管（SBV準拠）
- 否認理由をユーザーへ明確に提示

### FR-003 — 銀行口座連携
**優先度:** P0 · **出典:** US-003
- ベトナム主要10行に対応：Vietcombank・BIDV・VietinBank・Agribank・MB Bank・Techcombank・ACB・VPBank・TPBank・Sacombank
- Sepayマイクロデポジット（1,000〜3,000VND送金、ユーザーが金額を確認）による所有権検証
- マスク済み口座番号を保管し、Sepay経由でトークナイズ

**受入基準:**
- 検証時間60秒以内
- ユーザーあたり最大3口座まで連携可能
- 連携解除可能（24時間の猶予期間あり）

### FR-004 — ウォレットチャージ
**優先度:** P0 · **出典:** US-004
- 連携済み銀行口座からのチャージ
- 最低：10,000VND、最高：1回あたり50,000,000VND
- 1日上限：100,000,000VND

**受入基準:**
- 30秒以内にウォレットへ反映
- 成功 / 失敗時に通知配信
- Idempotency-Keyヘッダによる冪等性確保

### FR-005 — P2P送金（電話番号）
**優先度:** P0 · **出典:** US-005
- 他のVietPayユーザーへ電話番号でVND送金
- 最低：10,000VND、最高：1取引あたり10,000,000VND
- 1日上限：100,000,000VND
- 任意メモ（256文字）

**受入基準:**
- 受取人電話番号がVietPayユーザーであることを検証
- 生体認証またはPINによる確認必須
- 冪等性必須（Idempotency-Key必須）
- E2Eレイテンシ3秒以内
- 受取人へのプッシュ通知5秒以内
- 受取人が「受取確定」していなければ30秒以内にキャンセル可能

### FR-006 — P2P送金（銀行口座）
**優先度:** P0 · **出典:** US-006
- VietPay外の銀行口座へVND送金
- 上限はFR-005と同様
- Sepay（NAPAS 24/7）経由でルーティング

**受入基準:**
- 30秒以内に決済完了（営業時間内）、5分以内（営業時間外）
- 手数料：月3回まで無料、以降1回あたり5,000VND
- レシートPDFのダウンロード可能

### FR-007 — VietQR生成（受取側）
**優先度:** P0 · **出典:** US-007
- 静的QR生成（ウォレットに紐付け、金額なし）
- 動的QR生成（金額 + メモ + 15分の有効期限を指定）
- フォーマット：Napas準拠のVietQR EMV規格

**受入基準:**
- 他行アプリ（Vietcombank・BIDVなど）でスキャン可能
- 確認用に金額 + メモを明確に表示
- 動的QRは有効期限切れ時に自動更新

### FR-008 — VietQRスキャン（支払側）
**優先度:** P0 · **出典:** US-008
- カメラからQRをスキャン
- 加盟店情報 + 金額をパース
- 確認画面 → 生体認証 → 実行

**受入基準:**
- カメラ権限フローが明確
- フラッシュモード対応（暗所用）
- カメラ不調時の手動入力フォールバック
- 静的QR（金額なし）に対する金額編集

### FR-009 — トランザクション履歴
**優先度:** P0 · **出典:** US-009
- 全トランザクションを新着順で一覧表示
- フィルタ：日付範囲・種別（入金/出金）・カテゴリ
- 検索：電話番号 / 金額 / メモ
- 詳細表示：取引情報全体 + レシート

**受入基準:**
- カーソルベースのページネーション、20件/ページ
- 初回ロード500ms以内
- 日付範囲のCSVエクスポート（最大90日間）
- 加盟店データベース + MLによる自動カテゴリ分類

### FR-010 — プッシュ通知
**優先度:** P0 · **出典:** US-010
- 通知対象：受取・送金成功・チャージ完了・セキュリティアラート
- FCM（Android）+ APNs（iOS）
- アプリ内インボックスに90日間保管
- 通知設定（種別ごとの細かなトグル）

**受入基準:**
- 配信成功率95%以上、30秒以内
- ベトナム語 + 英語のローカライゼーション
- 関連画面へのディープリンク

### FR-011 — セキュリティ：生体認証
**優先度:** P0 · **出典:** US-011
- Face ID / Touch ID（iOS）· 指紋認証 / Face Unlock（Android）
- 500,000VND以上の全トランザクションで必須
- 24時間アイドル後のログインで必須
- 生体認証3回失敗時はPINにフォールバック

**受入基準:**
- オンボーディングで登録（任意、スキップ可）
- 端末変更時に再登録
- 端末がjailbreak / root済みの場合は無効化

### FR-012 — アカウント設定
**優先度:** P0 · **出典:** US-012
- PIN変更（現在のPIN検証付き）
- 連携銀行口座の管理
- 通知設定
- ログアウト / 全端末からログアウト
- アカウント削除（30日の猶予期間。KYCデータはSBV準拠で5年保管）

**受入基準:**
- PIN変更時に通知 + メール確認を送信
- 全端末ログアウトは5秒以内に全セッションをクリア
- アカウント削除フローは3段階の確認 + メール

## 3. 非機能要件

### NFR-001 — パフォーマンス
- API p50レイテンシ100ms未満、p95 200ms未満、p99 500ms未満
- アプリコールドスタート2秒未満（iOS）、3秒未満（Android）
- トランザクション履歴ロード：初回500ms未満、ページング200ms未満

### NFR-002 — 可用性
- SLA 99.9%稼働率（月間ダウンタイム43.8分まで許容）
- マルチAZデプロイ、RTO 5分未満、RPO 1分未満
- グレースフルデグラデーション：Sepayダウン時はキュー + リトライ + ユーザー通知

### NFR-003 — スケーラビリティ
- ローンチ時に10万同時ユーザーを想定
- 再アーキテクチャ不要で1M MAUまで水平スケール
- データベース：リードレプリカ + コネクションプーリング

### NFR-004 — セキュリティ
- クライアント・サーバー間TLS 1.3、サービス間TLS 1.2以上
- 保管時暗号化：AES-256-GCM（DB・S3・Redisスナップショット）
- シークレットはAWS KMS経由、ハードコード禁止
- レート制限：認証済みユーザー100 req/min、未認証10 req/min
- OWASP Top 10監査を四半期ごとに実施
- ローンチ前にペネトレーションテスト実施

### NFR-005 — コンプライアンス
- SBV通達23/2019 + 17/2024（電子ウォレットライセンス）
- AML報告：単発400M VND以上、または1日合計600M以上のトランザクションを自動フラグ
- PCI-DSS Level 1（Sepayトークナイゼーション経由、PAN非保持）
- 個人情報保護法 NĐ 13/2023
- KYC保管期間5年
- PIIアクセスの全監査ログ（不変、S3 Object Lock保管）

### NFR-006 — ユーザビリティ
- ベトナム語を主言語、英語を副言語
- オンボーディング5分以内に完了可能（目標3分以内）
- WCAG 2.1 Level AA準拠
- iOS 14以上およびAndroid 8以上をサポート（市場の95%をカバー）

### NFR-007 — 信頼性
- 全ミューテーションAPIで冪等性確保（Idempotency-Key必須）
- Kafka経由のイベント配信はat-least-once、コンシューマ側で重複排除
- DataDog APMによる分散トレーシング
- クラッシュフリー率99.5%以上（Sentry）

### NFR-008 — 保守性
- コードカバレッジ：行80%以上、ブランチ70%以上
- ファイルサイズ200 LOC未満、関数50 LOC未満
- Conventional Commits、semver
- 重要な意思決定はADR（Architecture Decision Records）で記録

## 4. 制約条件

- **予算:** バックエンド8名 + モバイル4名 + デザイナー2名 + PM 1名 + QA 1名 × 12週間
- **タイムライン:** 内部ベータ第12週、パブリックローンチ第16週
- **技術的負債:** 既存コードベースの継承なし、ゼロから構築
- **ベンダーロックイン許容度:** AWS可（インフラ）、Sepay可（フォールバックプラン有）、VNPT eKYC可（代替FPTを特定済み）

## 7. 用語集

| 用語 | 定義 |
|------|------|
| MAU | Monthly Active Users（月間アクティブユーザー） |
| ウォレット | VietPay内ユーザーのVND元帳アカウント |
| Idempotency Key | トランザクション単位のユニークUUID。リトライ時の二重支払いを防止 |
| VietQR | ベトナムQR決済規格、Napas運営 |
| Sepay | 決済アグリゲータ統合プラットフォーム |
| eKYC | 電子的本人確認（CMND/CCCDスキャン + リブネス検証） |
| CMND | Chứng Minh Nhân Dân（旧式IDカード） |
| CCCD | Căn Cước Công Dân（チップ搭載新式IDカード） |
| SBV | State Bank of Vietnam（ベトナム国家銀行） |
| Napas | National Payment Services VN（銀行間スイッチ） |
